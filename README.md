## 2. 当前整理方向

目前这个 Wiki 目录主要采用“系列目录页 + 单作条目页”的组织方式：

- 系列目录页负责梳理作品关系、阅读顺序与当前覆盖范围
- 单作条目页负责展开作品概览、剧情、玩法、角色、历史定位与参考资料

这种做法的好处是：

- 目录结构清楚
- 适合在 Gitea 中直接浏览
- 后续增补单作时不需要改动太多整体结构
- 适合逐步扩展成更完整的游戏资料库

## 3. 后续可继续补充的方向

如果后续继续扩写这个 Wiki，比较适合的方向包括：

- 补更多系列目录页
- 补已有系列中尚未覆盖的单作
- 增加角色、势力、地图、模式等专题条目
- 增加时间线、世界线与版本演化整理页
- 为每个系列单独建立更完整的索引与导航结构

## 4. 推荐使用方式

如果只是想快速找某个系列的内容，最适合直接从上面的系列目录页进入。

如果后续条目越来越多，也可以继续保持现在这套层级：

1. `Wiki/README.md` 作为总入口
2. 各系列目录中的 `README.md` 作为系列入口
3. 各单作 Wiki 文件作为具体内容页
