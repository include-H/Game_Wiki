参考仓库内 `Wiki/WIKI_WRITING_GUIDE.md`，参考互联网公开资料、官方资料与相关 Wiki，为【某个系列】撰写中文游戏 Wiki。

系列名称：
[例如：合金装备系列]

执行方式：
1. 一次只处理这一个系列，不要混入其他系列内容。
2. 先创建或更新该系列的 `README.md`，用于说明：
   - 系列概览
   - 分期方式
   - 阅读顺序
   - 当前已覆盖条目
   - 版本关系与补充说明
3. 再按系列内部阶段分批写单作 Wiki。
4. 每一批建议只写 2 到 4 作，写完后同步更新系列目录。

单作要求：
1. 每一作必须独立核对以下信息：
   - 正式名称
   - 常见中文译名
   - 开发商 / 发行商
   - 发售日期
   - 平台
   - DLC / 资料片 / 重制版 / 导演剪辑版 / 整合版关系
2. 统一采用以下结构：
   - 标题
   - 开头说明
   - 引言
   - 作品概览
   - 基础信息速览
   - 世界观与背景设定
   - 剧情概要
   - 玩法与结构特色
   - 主要角色与势力
   - 评价与历史定位
   - 一句话总结
   - 参考资料
3. 文风保持仓库现有“中文整理型 Wiki”风格：
   - 中文自然
   - 有整理感
   - 不写成百科资料堆
   - 不写成宣传文案
4. 如果某作存在重要 DLC、资料片、加强版、重制版或合集：
   - 需要写清与本体关系
   - 对剧情或完整体验有明显意义的内容要并入正文说明
   - 纯服装、纯数值、纯附赠内容可以简写

防止上下文污染的要求：
1. 只讨论当前系列。
2. 每写一作前，先明确这作在系列中的独特定位，再落正文。
3. 不要把前一作的开发信息、版本关系、剧情设定套用到后一作。
4. 同阶段作品可以共享系列背景，但具体事实必须逐作核对。

输出要求：
- 直接在仓库内创建或更新 Markdown 文件
- 先列出本批次准备创建的文件
- 然后按“系列 README -> 单作 1 -> 单作 2 -> 单作 3 ...”顺序执行
- 完成后同步更新总目录与系列目录
