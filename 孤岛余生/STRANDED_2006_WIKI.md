# 《Stranded》Wiki

> 说明：本文为面向中文读者整理的《Stranded》Wiki 初稿，主要依据 Pocket Gamer 的首发新闻与评测、以及公开资料整理而成。本文包含剧情概述、玩法结构与一定剧透。本文讨论的主体是 2006 年由 Glu Mobile 推出的 Java 手机游戏《Stranded》，它也是中文玩家常误称为“孤岛余生”的一条重要来源。

## 1. 开头说明

这篇条目讨论的是 Glu Mobile 的《Stranded》，不是 G5 / Awem 的 `The Island: Castaway`。两者在中文语境里经常共用“孤岛余生”这个名字，但正式英文名、发行体系与后续续作都不同。

## 3. 作品概览

《Stranded》是由 Glu Mobile 开发并发行的 Java 手机冒险游戏，于 2006 年 8 月前后进入公开市场。它在系列中的独特定位，是“把 Glu 这条孤岛生存冒险线第一次完整立住的原点作”。如果只看结构，你会发现它和后来很多人熟悉的“孤岛余生”题材作品非常接近：一群海难幸存者被困在神秘岛屿上，玩家需要通过探索、采集、捕鱼、狩猎、种植、烹饪与任务推进来维持生存，并逐步揭开岛上的秘密。

但《Stranded》真正特别的地方，不只是题材抓人，而是它在 2006 年的手机平台条件下，已经把“开放感”“日夜变化”和“生活劳动”这些东西揉得相当自然。Pocket Gamer 当年的报道和评测都反复提到，它不只是做一个线性闯关的冒险游戏，而是让玩家在完成主线之余，自由安排捕猎、钓鱼、种菜和做饭的节奏，甚至让现实时间部分影响游戏里的时间流动。

这也解释了为什么它在很多老玩家印象里分量很重。它不是最复杂的生存游戏，也不是最强叙事的剧情冒险，但在功能机时代，它已经能给人一种“手机里居然装着一个完整小世界”的感觉。很多人后来再看到 G5 的 `The Island: Castaway`，会自然地把两者并在一起记忆，本质上也是因为初代《Stranded》已经把这类体验的轮廓勾得很鲜明。

从历史位置看，《Stranded》也是 2006 年前后 Java 手机原创冒险游戏中相当有代表性的一作。Pocket Gamer 对它的评价很高，甚至把它看作当年最出色的移动冒险游戏之一。今天回头看，它的体量、操作和画面都带着时代限制，但它的设计心气并不小。

## 5. 世界观与背景设定

《Stranded》的开场设定很直接，也很有效。玩家从一场海难中幸存下来，醒来时已经身处陌生岛屿，周围还有同样被困的陌生人。最表层的任务当然是活下去和离开这里，但游戏很早就暗示，这地方并不是单纯的无人荒岛。岛上有不寻常的威胁、难以解释的秘密，也有并不完全值得信任的人。

这种设定让《Stranded》的气质一直维持在“轻生存 + 轻悬疑”的位置上。它没有把世界观写成硬核灾难模拟，也没有走彻底恐怖路线，而是用连续不断的小任务、生活劳动和章节推进，慢慢把神秘感裹进去。玩家不是通过大量过场动画理解这座岛，而是通过每天做的事情，一点点靠近真相。

游戏的另一个重要背景特征，是时间。Pocket Gamer 在首发报道里提到，本作有一套 24 小时机制，会影响故事触发和环境变化。也正因为这一点，岛屿不只是地图背景，更像一个会随着时间变脸的空间。

## 6. 剧情概要

### 6.1 海难与幸存者群体

主角在破损救生艇旁醒来，确认自己和其他幸存者一起被困在荒岛上。最早阶段的重点不是伟大谜题，而是活命：收集材料、生火、获得食物，并让这群人至少先有一个能维持下去的基本秩序。

### 6.2 从跑腿生存到岛屿探索

随着最初危机稍微稳定，玩家开始围着岛上的各个角色接任务。表面上看，这些任务大多是收集、制作和跑腿，但它们不断把玩家往更远的区域推，也让人逐渐察觉这里还有更危险、更古怪的东西。

### 6.3 秘密、反转与章节推进

Pocket Gamer 的评测提到，本作有完整的章节结构，而且每章结尾都像电视剧式的悬念收束。也就是说，它并不是一部纯生活模拟，而是一部确实在认真讲故事的荒岛冒险游戏。岛屿秘密、潜在反派与逃离希望，会在这个过程中一点点浮出来。

### 6.4 生存劳动本身就是叙事的一部分

《Stranded》的剧情不靠长篇对白压人，而是让劳动和推进本身承担叙事作用。你在捕猎、钓鱼、种菜和做饭的时候，实际上也在把这个故事一点点往前推。因此，它的剧情节奏看似慢，其实黏性很强。

## 7. 玩法与结构特色

### 7.1 功能机时代少见的开放感

Pocket Gamer 在评测里特别强调，《Stranded》在手机游戏中显得相当开放。玩家虽然有主线任务要做，但并不会被彻底锁死在“做完 A 才能去 B”的窄轨上。你可以选择花时间抓野猪、钓鱼、种菜或先推进任务，这种自由度在当时很有吸引力。

### 7.2 生存循环不是点缀

这作的捕鱼、狩猎、种植和烹饪，并不只是小游戏性质的填充，而是维持能量、获得资源和继续探索的基础。食物既是生存必需品，也能拿来交易或支持更高效的工具升级。

### 7.3 真实时间与昼夜机制

初代最有辨识度的机制之一，是现实时间会部分映射到游戏内。不同时间段会出现不同的鱼类、敌人和秘密，这让荒岛有了一种“你不玩时它也还在继续运作”的感觉。

### 7.4 轻解谜与轻动作并存

《Stranded》不是高难动作游戏，也不是纯文字解谜。它更接近一种综合型冒险：一点解谜，一点资源管理，一点探索，一点动作，再加上一点悬疑叙事。也正因为平衡得比较温和，它才会让很多玩家觉得容易沉进去。

## 8. 主要角色与势力

### 8.1 主要角色

- 主角：海难后的幸存者，也是玩家理解整座岛的主要视角。
- 幸存者群体：提供任务、资源需求与人际张力，也是故事推进的重要来源。
- 岛上的潜在敌对者：让剧情从单纯求生转向带有悬念与反转的冒险。

### 8.2 主要势力

- 海难幸存者：代表最直接的生存需求与逃生愿望。
- 岛屿环境与野生生物：构成持续性的资源压力和危险来源。
- 岛屿秘密相关力量：包括隐藏威胁、谜团和剧情里的关键转折因素。

## 9. 评价与历史定位

《Stranded》在当年的口碑相当不错。Pocket Gamer 的首发评测把它看作 2006 年最出色的手机游戏之一，认可它把剧情、解谜和生存劳动混得很顺。尤其在那个时代，很多手机游戏还停留在短流程和单一机制上，而《Stranded》已经在尝试做一部更完整、更像“手机上的小型冒险世界”的作品。

它的优势主要在于题材抓人、系统组合新鲜、时间机制有记忆点，且整体流程很容易让人一路玩下去。它的局限也同样明显，比如动作和解谜难度都不算高，系统深度有限，本质上仍然受制于当年功能机平台的体量与输入方式。

但从历史位置来看，这些并不妨碍它成为中文“孤岛余生”记忆里很重要的一块拼图。很多后来被套到别的作品身上的荒岛生存印象，追根究底，其实正是从《Stranded》这类 Java 时代作品里长出来的。

## 10. 一句话总结

《Stranded》最值得记住的，是它在 Java 手机时代就把荒岛求生、资源管理和悬疑冒险捏成了一部很完整的小型作品。
